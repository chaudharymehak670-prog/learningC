OBJECT ORIENTED PROGRAMMING (OOP)

Object-Oriented Programming (OOP) ek programming approach hai jisme programs ko Objects aur Classes ki help se design kiya jata hai. OOP code ko reusable, secure aur easy to maintain banata hai. Modern software development me OOP ka bahut important role hai.

Object-Oriented Programming (OOP) is a programming approach that uses classes and objects to design programs. It helps make code reusable, secure, organized, and easier to maintain.

What is OOP?

OOP ek programming concept hai jisme real-world objects ko Classes aur Objects ke through represent kiya jata hai. Har object ke paas apni properties aur functions hote hain.

OOP is a programming concept that represents real-world entities using classes and objects. Every object has its own properties and behaviors.

💡 Real Life Example

Socho ek Car hai.

Car ka Color, Model, Speed uski properties hain.

Car ka Start(), Stop(), Brake() uske functions hain.

Isi tarah OOP me har object ke paas data aur functions dono hote hain.

Why Do We Use OOP?

OOP ka use bade programs ko organize karne aur code ko reusable banane ke liye kiya jata hai.

OOP helps organize large programs, improves code reusability, and makes software easier to develop and maintain.

  • Reduce code duplication.
  • Improve code reusability.
  • Increase security.
  • Make programs easier to maintain.
  • Represent real-world objects easily.

Main Concepts of OOP

OOP mainly chhe important concepts par based hai.

Object-Oriented Programming is mainly based on six important concepts.

Concept Description
Class Blueprint for creating objects.
Object Instance of a class.
Encapsulation Combines data and functions.
Inheritance Allows code reusability.
Polymorphism Same function behaves differently.
Abstraction Hides unnecessary details.

Class

Class ek blueprint ya template hota hai jiske basis par objects create kiye jaate hain.

A class is a blueprint or template used to create objects. 

class Student
{
public:

    string name;

};
💡 Best Use

Class ka use similar objects ko organize karne ke liye kiya jata hai.

Object

Object class ka real instance hota hai. Object ke through hum class ke members ko access karte hain.

An object is a real instance of a class used to access its data members and member functions. 

Student s1;
💡 Best Use

Ek class se multiple objects create kiye ja sakte hain.

Class and Object Example

Neeche diya gaya program Class aur Object ka basic example dikhata hai.

The following program demonstrates the basic use of a class and an object.

View Example 
#include <iostream>
using namespace std;

class Student
{
public:

    string name;

    void display()
    {
        cout << "Name = " << name;
    }

};

int main()
{

    Student s1;

    s1.name = "Anjali";

    s1.display();

    return 0;

}
Output

Name = Anjali

Explanation
A class named Student is created.
An object s1 is created from the class.
The value "Anjali" is stored in name.
The display() function prints the value.

Data Members

Class ke andar jo variables declare kiye jaate hain unhe Data Members kehte hain.

Variables declared inside a class are called data members. 

class Student
{
public:

    string name;

    int age;

};
💡 Best Use

Data Members object ki information store karte hain jaise Name, Age, Salary, Roll Number etc.

Member Functions

Class ke andar jo functions likhe jaate hain unhe Member Functions kehte hain.

Functions declared inside a class are called member functions. 

class Student
{
public:

    void show()
    {
        cout << "Welcome";
    }

};
💡 Best Use

Member Functions object ke data par operations perform karte hain.

Access Specifiers

Access Specifiers decide karte hain ki class ke members ko kahan se access kiya ja sakta hai.

Access specifiers define the accessibility of class members.

Specifier Description
public Accessible from anywhere.
private Accessible only inside the class.
protected Accessible inside the class and derived classes.

Access Specifier Example

Ye program public access specifier ka use dikhata hai.

This program demonstrates the use of the public access specifier.

View Example 
#include <iostream>
using namespace std;

class Student
{
public:

    int marks = 90;

};

int main()
{

    Student s1;

    cout << s1.marks;

    return 0;

}
Output

90

Explanation
The data member marks is declared as public.
Therefore, it can be accessed directly using the object.
The value 90 is displayed.

Constructor

Constructor ek special member function hota hai jo object create hote hi automatically call ho jata hai. Iska naam hamesha class ke naam jaisa hi hota hai.

A constructor is a special member function that is automatically called when an object is created. Its name is always the same as the class name. 

class Student
{
public:

    Student()
    {
        cout << "Constructor Called";
    }

};
💡 Best Use

Constructor object ko initialize karne ke liye use kiya jata hai.

Constructor Example

Ye program constructor ke automatic execution ko dikhata hai.

This program demonstrates how a constructor is called automatically when an object is created.

View Example 
#include <iostream>
using namespace std;

class Student
{
public:

    Student()
    {
        cout << "Object Created";
    }

};

int main()
{

    Student s1;

    return 0;

}
Output

Object Created

Explanation
An object s1 is created.
The constructor is automatically called.
Therefore, "Object Created" is displayed.

Destructor

Destructor ek special member function hota hai jo object destroy hone par automatically call hota hai.

A destructor is a special member function that is automatically called when an object is destroyed. 

class Student
{
public:

    ~Student()
    {
        cout << "Object Destroyed";
    }

};
💡 Best Use

Destructor memory release aur cleanup tasks ke liye use hota hai.

Encapsulation

Encapsulation ka matlab data aur functions ko ek hi class ke andar combine karna aur data ko direct access se protect karna hota hai.

Encapsulation is the process of combining data and functions into a single unit while protecting the data from direct access. 

class Student
{
private:

    int marks;

public:

    void setMarks(int m)
    {
        marks = m;
    }

    int getMarks()
    {
        return marks;
    }

};
💡 Best Use

Encapsulation improves data security and prevents unauthorized access.

Inheritance

Inheritance ek class ko doosri class ki properties aur functions use karne ki permission deta hai. Isse code reuse hota hai.

Inheritance allows one class to acquire the properties and functions of another class, improving code reusability. 

class Animal
{
public:

    void sound()
    {
        cout << "Animal Sound";
    }

};

class Dog : public Animal
{

};
💡 Best Use

Inheritance is useful when multiple classes share common properties and functions.

Inheritance Example

Ye program inheritance ka basic example dikhata hai.

This program demonstrates basic inheritance in C++.

View Example 
#include <iostream>
using namespace std;

class Animal
{
public:

    void sound()
    {
        cout << "Animal Sound";
    }

};

class Dog : public Animal
{

};

int main()
{

    Dog d;

    d.sound();

    return 0;

}
Output

Animal Sound

Explanation
The class Dog inherits from Animal.
The object d can directly use the sound() function.
Therefore, "Animal Sound" is displayed.

Polymorphism

Polymorphism ka matlab hai "One Name, Many Forms". Isme ek hi function alag-alag situations me alag behavior dikhata hai.

Polymorphism means "One Name, Many Forms". The same function can perform different tasks depending on the situation. 

class Animal
{
public:

    void sound()
    {
        cout << "Animal Sound";
    }

};

class Dog : public Animal
{
public:

    void sound()
    {
        cout << "Bark";
    }

};
💡 Best Use

Polymorphism is useful when different objects perform the same action in different ways.

Polymorphism Example

Ye program method overriding ki help se Polymorphism ko dikhata hai.

This program demonstrates polymorphism using method overriding.

View Example 
#include <iostream>
using namespace std;

class Animal
{
public:

    void sound()
    {
        cout << "Animal Sound";
    }

};

class Dog : public Animal
{
public:

    void sound()
    {
        cout << "Bark";
    }

};

int main()
{

    Dog d;

    d.sound();

    return 0;

}
Output

Bark

Explanation
The Dog class overrides the sound() function.
When d.sound() is called, the Dog class function executes.
Therefore, "Bark" is displayed.

Abstraction

Abstraction ka matlab sirf important information dikhana aur unnecessary details ko hide karna hota hai.

Abstraction means showing only the essential information while hiding unnecessary implementation details. 

class Shape
{
public:

    virtual void draw() = 0;

};
💡 Best Use

Abstraction is useful when users need to know what an object does, not how it does it.

Common Mistakes

  • Creating objects without understanding the class structure.
  • Using private members directly outside the class.
  • Confusing constructors with normal member functions.
  • Forgetting inheritance syntax.
  • Using OOP concepts where they are not required.

Important Note

Har class ka ek specific purpose hona chahiye. Chhoti aur well-designed classes code ko readable aur maintainable banati hain.

Each class should have a single responsibility. Small and well-designed classes improve readability and maintainability.

Did You Know?

Java, C++, C#, Python aur bahut si modern programming languages OOP concepts ko support karti hain. Most real-world software applications OOP par based hote hain.

Most modern software applications are built using Object-Oriented Programming because it improves code organization and reusability.
